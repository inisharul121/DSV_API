### 1. Build Everything
```bash
cd /Users/nisharul/Downloads/Workplace/DSV_API/monolith
npm run install-all
npm run build
```

### 2. Test Locally
```bash
cd backend
npm start
```
Visit: `http://localhost:3000`

---

## cPanel Upload & Deployment

### 1. Prepare for Upload
- [ ] Delete `node_modules/` folders (they're large)
- [ ] Ensure `backend/.env` exists with production config
- [ ] Verify `backend/public/` has `index.html` and other React files

### 2. Upload to cPanel
- [ ] Compress monolith folder: `zip -r monolith.zip monolith/ -x "*/node_modules/*"`
- [ ] Use cPanel File Manager to upload
- [ ] Extract in your public_html or home directory

### 3. Run Setup Script
On cPanel Terminal (SSH):
```bash
cd monolith
bash setup-server.sh
```
Or if setup-server.sh fails, run manually:
```bash
npm run install-all
npm run build
```

### 4. Configure cPanel Node.js App
In cPanel > Node.js Selector:
- **Application root:** `/path/to/monolith/backend`
- **Application startup file:** `src/app.js`
- **Listening port:** `3000` (or your choice)
- **Application mode:** `production`

### 5. Start the App
- Click **Run** in cPanel Node.js interface
- Wait 30 seconds for startup
- Visit your domain: `https://yourdomain.com`

## Troubleshooting

### Frontend not loading
1. Check `backend/public/index.html` exists
2. Check browser DevTools > Network tab
3. Verify backend is running

### API calls failing  
1. Check `backend/.env` variables
2. Verify database connection
3. Check cPanel error logs

### 502 Bad Gateway
1. Backend crashed - check logs
2. Wrong port configured
3. Node.js app not running - restart via cPanel

---

## Update Process

When you make changes:

1. Update source files in `frontend/src/` or `backend/src/`
2. Rebuild: `npm run build`
3. Upload new `backend/public/` folder
4. Restart Node.js app in cPanel
