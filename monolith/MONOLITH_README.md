# DSV API Monolith - cPanel Deployment Guide

This folder contains the monolith structure for deploying your DSV API application to cPanel hosting.

## Structure

```
monolith/
├── backend/              # Express backend application
│   ├── src/
│   ├── public/          # Static files (frontend build will be copied here)
│   ├── package.json
│   └── ...
├── frontend/            # React frontend application
│   ├── src/
│   ├── package.json
│   └── ...
├── build.js             # Build script for deployment
├── package.json         # Root package.json
└── MONOLITH_README.md   # This file
```

## Setup Instructions

### 1. Install Dependencies

```bash
npm run install-all
```

This will install dependencies for the root, backend, and frontend.

### 2. Build and Prepare for Deployment

```bash
npm run build
```

This script will:
- Install frontend dependencies
- Build the React frontend
- Copy all build files to `backend/public/`

### 3. Update Backend to Serve Frontend

Make sure your `backend/src/app.js` has this configuration:

```javascript
// Serve static files from public directory
app.use(express.static(path.join(__dirname, '../public')));

// API routes
app.use('/api', apiRoutes);

// Serve SPA - must be after API routes
app.get('*', (req, res) => {
  res.sendFile(path.join(__dirname, '../public/index.html'));
});
```

### 4. Test Locally

Before deploying:

```bash
cd backend
npm start
```

Then visit `http://localhost:PORT` to verify both API and frontend work correctly.

### 5. Deploy to cPanel

#### Option A: Upload Entire Monolith Folder
1. Compress the `monolith` folder (excluding node_modules)
2. Upload to cPanel via File Manager
3. Extract and run:
   ```bash
   npm run install-all
   npm run build
   ```

#### Option B: Deploy Only Necessary Files
1. Upload only:
   - `backend/` folder
   - `monolith/package.json` (optional, for scripts)
2. On server:
   ```bash
   npm run install-all
   npm run build
   ```

### 6. Deployment Checklist

- [ ] Environment variables configured (`.env` file in backend)
- [ ] Database connection verified
- [ ] Frontend built and copied to `backend/public/`
- [ ] Backend listening on correct port (usually 3000 or 5000)
- [ ] cPanel Node.js app configured to run backend
- [ ] SSL certificate installed (if using HTTPS)
- [ ] API endpoints accessible from `http://yourdomain.com/api/*`
- [ ] Frontend accessible from `http://yourdomain.com`

## Key Changes Made to Backend

**No code changes required!** The build script automatically:
- Builds the frontend
- Copies files to `backend/public/`
- Maintains existing backend structure

Your existing `backend/src/app.js` just needs to serve static files (see Step 3).

## Rebuilding After Frontend Changes

If you make changes to the frontend:

1. Rebuild:
   ```bash
   npm run build
   ```

2. Redeploy just the rebuilt `backend/public/` folder to cPanel

## Environment Variables

Keep your `.env` file in the `backend/` folder with:
```
NODE_ENV=production
PORT=3000
DATABASE_URL=your_database_url
DSV_API_KEY=your_key
```

## Troubleshooting

### Build fails
- Ensure Node.js version >= 14 is installed
- Check that `frontend/package.json` has a `build` script
- Verify all dependencies are installed: `npm run install-all`

### Frontend not loading
- Check `backend/public/` has an `index.html` file
- Verify Express is serving static files correctly
- Check browser console for API endpoint errors

### API calls failing
- Ensure API routes in `backend/src/routes/` are registered before the catch-all route
- Check CORS settings if frontend is on different domain
- Verify environment variables are set on cPanel

## Notes

- Keep `node_modules/` out of version control (already in .gitignore)
- The `frontend/dist/` folder is generated by the build script
- Original `backend/` and `frontend/` folders remain unchanged
- For updates, modify source files and run `npm run build` again

Happy deploying! 🚀
